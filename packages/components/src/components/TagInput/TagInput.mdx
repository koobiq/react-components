# TagInput

<Status variant="draft" />

An input component that allows users to enter, display, and manage multiple
tags or keywords dynamically. It supports adding and removing tags.

## Import

```tsx
import { TagInput } from '@koobiq/react-components';
```

## Controlled / uncontrolled

For simple cases, pass `defaultValue` and let the component manage tags
internally. For controlled mode, pass `value` and an `onChange` handler that
updates external state.

`inputValue` / `defaultInputValue` / `onInputChange` work the same way for the
text input.

<Story of={Stories.Controlled} />

## Splitting input

Tags are added when the user presses <kbd>Enter</kbd> or types a split
character. The default split pattern is `,` — to also split on `;` or
whitespace, pass a custom `splitPattern`. Pasting a delimited string into
the field creates several tags at once, leaving any text already typed in
the input intact.

When the field loses focus with a non-empty input value, the component
tries to commit it as a final tag (commit-on-blur).

## Form field

`TagInput` is a full form-field on par with `Input`. It accepts a visible
or hidden `label`, a helper `caption` below the control, and an
`errorMessage` paired with `isInvalid`. Mark the field as required with
`isRequired`, stretch it to the container with `fullWidth`, or change the
label position via `labelPlacement` and `labelAlign`.

Validation is driven from the outside: both `isInvalid` and `errorMessage`
can be controlled by your own state. In the example below the second
field stays invalid until at least one tag has been added — once a tag
is committed, the error gives way to a helper caption.

<Story of={Stories.FormField} />

## Disabled

When `isDisabled` is set, the input is disabled and every tag renders in a
disabled state. The remove button on each tag stays visible but cannot be
clicked, and the cleaner is hidden.

<Story of={Stories.Disabled} />

## Read-only

When `isReadOnly` is set, the input is focusable for navigation but no
modifications are allowed — typing is blocked and the remove buttons on
tags are not rendered.

<Story of={Stories.ReadOnly} />

## Cleaner

Enable the cleaner button with `isClearable`. Pressing it removes all tags,
clears the text input, calls `onClear`, and returns focus to the input.

<Story of={Stories.Clearable} />

## Inside a Form

When wrapped in a `<Form>`, `TagInput` inherits `isDisabled`, `isReadOnly`,
`labelPlacement` and `labelAlign` from the form context.

## Accessibility

- The root field renders a `<label>` linked to the inner `<input>`. Pass
  either `label` or `aria-label` so the field has an accessible name.
- The tag list inside the field uses `role="grid"` when there are tags,
  `role="group"` otherwise. Each tag is a `role="row"` with one
  `role="gridcell"`.
- When `onChange` is wired up, the remove button on each tag (and the
  global cleaner) is announced with its localized `aria-label`.
- Keyboard hints for tag removal (<kbd>Backspace</kbd> / <kbd>Delete</kbd>)
  are exposed via `aria-describedby` on tags — only for keyboard /
  assistive modalities, not for pointer users.

## Keyboard

Inside the text input:

| Key                                             | Behavior                                                                          |
| ----------------------------------------------- | --------------------------------------------------------------------------------- |
| <kbd>Enter</kbd>                                | Commit the current input value as a new tag (if non-empty).                       |
| _split character_                               | Commit and reset the input — same effect as <kbd>Enter</kbd>.                     |
| <kbd>Backspace</kbd> / <kbd>ArrowLeft</kbd>     | If the input is empty and there are tags, focus the last tag.                     |
| <kbd>Ctrl</kbd> / <kbd>Cmd</kbd> + <kbd>A</kbd> | If the input is empty and there are tags, select all tags and focus the last one. |
| <kbd>Shift</kbd> + <kbd>Tab</kbd>               | If the input is empty and there are tags, move focus to the first tag.            |
| <kbd>Tab</kbd>                                  | Move focus to the cleaner button (if visible) or out of the field.                |

Inside a focused tag — see [TagList keyboard](?path=/docs/components-taglist--docs#keyboard) for the full table (<kbd>Arrow</kbd> navigation, <kbd>Space</kbd> selection, <kbd>Delete</kbd> / <kbd>Backspace</kbd> removal, <kbd>Ctrl</kbd> + <kbd>A</kbd>, etc.).
