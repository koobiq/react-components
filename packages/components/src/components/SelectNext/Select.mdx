# SelectNext

<Status variant="experimental" />

A select displays a collapsible list of options and allows a user to select one or more of them.

## Import

```tsx
import { SelectNext as Select } from '@koobiq/react-components';
```

## Content

### Dynamic items

Select accepts static and dynamic collections. The examples above show static collections,
which can be used when the full list of options is known ahead of time.
Dynamic collections, as shown below, can be used when the options come from an external
data source such as an API call, or update over time.

As seen below, an iterable list of options is passed to the `Select` using the `items` prop.
Each item accepts an `id` prop, which is passed to the `onChange` handler to identify
the selected item. Alternatively, if the item objects contain an id property, as shown
in the example below, then this is used automatically and an `id` prop is not required.

<Story of={Stories.Content} />

### Disabled options

Select supports marking items as disabled using the `disabledKeys` prop.
Each key in this list corresponds with the `id` prop passed to the `Select.Item` component.

<Story of={Stories.DisabledOptions} />

### Empty state

With `allowsEmptyCollection` enabled, the Select can be opened even when the collection is empty.
In this case, the component shows an empty-state message in the dropdown list.
You can customize this message — either as plain text or a component — using the `noItemsText` prop.

<Story of={Stories.NoItems} />

A select component can display items grouped together in sections.

### Item content

Select items can be composed with helper components:

- `Select.ItemText` for text and captions.
- `Select.ItemAddon` for icons, badges, or other secondary content.

Select items also have layout props, such as `align`, which sets vertical alignment. Use `align="start"` for captions or multi-line content.

## Behavior

### Single selection

Setting a selected option can be done by using the `defaultValue` or `value` prop.
The selected id corresponds to the `id` prop of an item.

<Story of={Stories.SingleSelection} />

### Multiple selection

You can use the `selectionMode="multiple"` prop to allow multiple selection.

<Story of={Stories.MultipleSelection} />

### Selected value tags

In `selectionMode="multiple"`, selected values are displayed as tags by default.

To control how tags are displayed, use the `selectedTagsOverflow` prop. It supports the following values:

- `responsive` (default) — tags collapse into a summary (e.g., "+3") to fit the available width without increasing the component block size.
- `multiline` — tags wrap to multiple lines, increasing the component height as needed.

<Story of={Stories.SelectedTagsOverflow} />

### Disabled

When the select component is disabled, it cannot be interacted with.

<Story of={Stories.Disabled} />

### Clear button

If you pass the `isClearable` prop to the select, it will have a clear
button which will be visible only when a value is selected.

<Story of={Stories.ClearButton} />

### Searchable

Select with search renders a search input at the top of the menu so users can quickly filter the available options.
Enable this behavior with the `isSearchable` prop. The input can be controlled via `inputValue` or used uncontrolled
with `defaultInputValue`, and the `onInputChange` callback is called whenever the search value is edited by the user.

<Story of={Stories.Searchable} />

### Open

#### Default open

Select isn't opened by default. The `defaultOpen` prop can be used to set the default state.

#### Controlled open

The `isOpen` prop can be used to make the opened state controlled.
The `onOpenChange` event is fired when the select's open state changes.

<Story of={Stories.Open} />

### Asynchronous loading

Select supports loading data asynchronously, and will display a progress circle when the `isLoading` prop is set.
It also supports infinite scrolling to load more data on demand as the user scrolls, via the `onLoadMore` prop.

<Story of={Stories.AsynchronousLoading} />
