# FileUpload

<Status variant="draft" />

FileUpload lets users choose files, see them in a list, and remove them. It
doesn't upload anything on its own — your app does the actual upload and tells
FileUpload what to show (progress or an error).

## Import

```tsx
import { FileUpload } from '@koobiq/react-components';
```

## Anatomy

By default FileUpload renders a ready-made layout. To customize it, build it
from these parts:

```tsx
<FileUpload>
  <FileUpload.List />
  <FileUpload.Dropzone>
    Drop files here or <FileUpload.Trigger />
  </FileUpload.Dropzone>
</FileUpload>
```

- `FileUpload.Dropzone` — the drop area with the browse link.
- `FileUpload.Trigger` — the link that opens the file picker.
- `FileUpload.List` — the list of chosen files.
- `FileUpload.Item` — one file: icon, name, size, and a remove button.

## Multiple files

## Compact

Use `size="compact"` in multiple mode when uploading is a minor part of the
form. The empty state shrinks to a small strip. Once files are added, it looks
the same as the default size.

## Drag and drop

Users can drag files straight onto the component. Anything that isn't a file
(like dragged text) is ignored.

## Scrolling

Long lists scroll. Set a max height on `FileUpload.List` to choose when.

<Story of={Stories.Scrollable} />

## Controlled

Pass `value` and `onChange` to control the list yourself. Do this when your app
needs to update a file — for example to show its upload progress or an error.

<Story of={Stories.Controlled} />

## Progress

While a file uploads, set `isLoading` on it to show a spinner. Add `progress`
(`0`–`100`) to show how far it's got, or leave it out for an endless spinner.
Users can still remove the file while it loads.

<Story of={Stories.UploadProgress} />

## Validation

FileUpload doesn't check files for you. Validate them in `onChange` — by size,
type, or any rule — then set `errorMessage` on any file that fails. That file
turns red. `accept` only filters the system file picker, so still check dropped
files yourself.

<Story of={Stories.Validation} />

## File size

Set `showFileSize` to show each file's size (on by default with
`allowsMultiple`). Sizes follow the app's language.

## Accessibility

- The browse link opens your system's file picker.
- Keyboard and screen-reader users get a hidden button to add files without a
  mouse.
- Each remove button says which file it removes.
- After removing a file with the button, <kbd>Delete</kbd>, or
  <kbd>Backspace</kbd>, focus moves to the next file — or to the browse link if
  the list is now empty.
- All text and size units follow the app's language.
