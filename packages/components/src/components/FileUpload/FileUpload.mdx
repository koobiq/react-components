# FileUpload

<Status variant="draft" />

FileUpload lets users choose files, see them in a collection, and request
removal. It doesn't upload anything or own item data on its own — your app keeps
the items and updates them from `onAdd` and `onRemove`.

## Import

```tsx
import { FileUpload } from '@koobiq/react-components';
```

## Data model

FileUpload follows the React Aria Collection Components API used by
Breadcrumbs: pass item objects to `items`, and render them with static children
or a `children(item)` function. FileUpload handles the file picker,
drag-and-drop, empty state, add-more area, and focus restoration. Your app owns
the collection:

```tsx
<FileUpload
  items={items}
  onAdd={(files) => setItems((prev) => [...prev, ...files.map(toItem)])}
  onRemove={(id) => setItems((prev) => prev.filter((item) => item.id !== id))}
>
  {(item) => (
    <FileUpload.Item id={item.id} textValue={item.name}>
      <FileUpload.ItemIcon />
      <FileUpload.ItemContent>
        <FileUpload.ItemName>{item.name}</FileUpload.ItemName>
        <FileUpload.ItemSize>{item.size}</FileUpload.ItemSize>
      </FileUpload.ItemContent>
      <FileUpload.RemoveButton />
    </FileUpload.Item>
  )}
</FileUpload>
```

## Anatomy

Customize the default UI with compound parts:

```tsx
<FileUpload
  items={items}
  onAdd={handleAdd}
  onRemove={handleRemove}
  renderEmptyState={() => (
    <FileUpload.Empty>
      <FileUpload.EmptyIcon />
      <FileUpload.EmptyTitle>Upload evidence files</FileUpload.EmptyTitle>
      <FileUpload.EmptyDescription>
        Drag files here or <FileUpload.Trigger>select files</FileUpload.Trigger>
      </FileUpload.EmptyDescription>
    </FileUpload.Empty>
  )}
  renderAddMore={() => (
    <FileUpload.AddMore>
      Add more files or <FileUpload.Trigger>select files</FileUpload.Trigger>
    </FileUpload.AddMore>
  )}
>
  {(item) => (
    <FileUpload.Item id={item.id} textValue={item.name}>
      <FileUpload.ItemIcon />
      <FileUpload.ItemContent>
        <FileUpload.ItemName>{item.name}</FileUpload.ItemName>
        <FileUpload.ItemSize>{item.size}</FileUpload.ItemSize>
      </FileUpload.ItemContent>
      <FileUpload.RemoveButton />
    </FileUpload.Item>
  )}
</FileUpload>
```

Replace only the part you need. For example, pass just `EmptyIcon` and FileUpload
will fill the default title and description.

## Progress

While a file uploads, render your loading state in the item row. `ItemIcon`
shows the default file icon when no children are passed, and custom content when
you provide it.

<Story of={Stories.UploadProgress} />

## Validation

FileUpload doesn't check files for you. Validate them in `onAdd` — by size,
type, or any rule — then set your item state. `accept` only filters the system
file picker, so still check dropped files yourself.

## Accessibility

- The browse link opens your system's file picker.
- Keyboard and screen-reader users get a hidden button to add files without a
  mouse.
- Each remove button says which file it removes.
- After removing a file with the button, <kbd>Delete</kbd>, or
  <kbd>Backspace</kbd>, focus moves to the next file — or to the browse link if
  the list is now empty.
- All text and size units follow the app's language.
