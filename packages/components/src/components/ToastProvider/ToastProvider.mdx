## Usage

The `ToastProvider` must be added to the application before using the `toast` function.
This is required to initialize the context for managing toasts.

```tsx
import { Provider, ToastProvider } from '@koobiq/react-components';

export function Layout({ children }: { children: React.ReactNode }) {
  return (
    <>
      <ToastProvider />
      {children}
    </>
  );
}
```

And next, just use the `toast.add()` function to show a toast.

```tsx
<Button onPress={() => toast.add({ title: 'Toast title' })}>
  Show a toast
</Button>
```

## Toast Queue

Toasts are shown in a stack and disappear **one by one**, so the list doesn't jump and is easy to read.

**How it works**:

- Each toast has **its own timeout** and will not close earlier than it should.
- Timed toasts close **in order**: the one that appeared first closes first (FIFO).
- After any toast closes, there is a **2-second** pause before the next one can close.
- **Manual close** (by button) also starts this 2-second pause.
- When the user **hovers or focuses** the toast area, all timers are paused and continue when the user leaves.
- Toasts **without a timeout** stay on screen until the user closes them.

For users this means: every toast lives as long as it should, but they **disappear smoothly, one after another**,
not all at once.
